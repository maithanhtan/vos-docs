Dashboard is the primary trading workspace. It consolidates order entry, market depth, chart context, and the order blotter.

## Dashboard layout actions

| Action | Result |
|---|---|
| Edit layout | Enables widget drag, resize, and remove controls. |
| Add Widget | Opens the widget picker for charts, depth, tickets, blotters, and supporting panels. |
| Add All / Remove All | Adds or removes the available dashboard widgets in one action. |
| Share layout | Copies the current layout definition. |
| Import layout | Opens a JSON import dialog and applies the layout only after validation. |
| Arrange / Reset | Reflows widgets or restores the default workspace layout. |

## Widgets and shared context

The widget picker contains the trading and monitoring panels available to the current user.

| Widget | Operating context |
|---|---|
| P&L Portfolio | Reviews portfolio performance for the selected account context. |
| Orders Workspace | Monitors active and historical orders. |
| Positions | Reviews holdings and live mark-to-market values. |
| Risk Monitor | Reviews cash, holdings, exposure, and risk-check status. |
| Order Ticket | Stages an order without leaving the Dashboard. |
| Reports | Reviews reporting and audit summaries available to the user. |
| Price Depth | Reviews the order book and recent trades for a symbol. |
| Trading Chart | Reviews candle and volume history for a symbol. |
| Market Indices | Reviews available market index values. |
| Price Board | Embeds the market price table in the Dashboard layout. |
| Latency Analytics | Reviews order timing and throughput for the selected period. |
| Operational Workspace | Embeds the account and operational view when permitted. |

Linked widgets can share symbol or account context. Unlink a widget before changing its context independently.

## Order Ticket

<div className="guide-split">
  <div className="guide-split-media">
    <img src="/images/vero-oms-ui/dashboard-order-ticket.png" alt="Dashboard order ticket" />
  </div>
  <div className="guide-split-copy">

The ticket contains account, symbol, order type, side, price, quantity, order summary, and buying power preview fields.

| Ticket action | Result |
|---|---|
| Account / symbol search | Updates the ticket context and refreshes market preview values. |
| Buy / Sell | Switches side and recalculates the order summary. |
| Order type | Changes the price fields required by the selected type. |
| Quantity stepper | Increases or decreases the staged quantity. |
| Submit | Sends the staged ticket only when required fields are valid. |
| Cancel / reset | Clears staged ticket state without submitting. |

  </div>
</div>
