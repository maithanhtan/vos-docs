![System workspace overview](/images/vero-oms-ui/system-full.png)

## Latency Overview

The latency overview contains filters, latency KPIs, latency vs throughput chart, and system load.

![System latency overview](/images/vero-oms-ui/system-latency-overview.png)

| Latency control | Behavior |
|---|---|
| Range selector | Switches between preset windows and resets manual range state. |
| From / To date-time | Sets a manual latency range. |
| Type / account / status | Filters visible latency rows by message type, account, and status. |
| Symbol / order search | Filters rows by symbol or order id. |
| Auto refresh 3s | Toggles periodic reload of the latency report. |
| Chart zoom | Sets a manual range from the selected chart window. |

## Users

Users displays user filters, user table, access overview, login history, and selected user panel. A data unavailable message appears when user information cannot be loaded.

![System users](/images/vero-oms-ui/system-users.png)

The Users workspace includes a text filter, user-state selector, refresh action, and **NEW USER** action. Selecting a table row updates the selected-user panel with identity, state, roles, allowed features, creation time, last login, login IP, and active session count.

![System users access overview](/images/vero-oms-ui/system-users-access-overview.png)

Each user row exposes action buttons for state change, password reset, 2FA reset, and access editing. The state button opens an activate/deactivate confirmation dialog. The password button opens a reset-password dialog that requires an 8-character minimum password. The 2FA button opens a confirmation before removing the current second-factor setup.

| User action | Behavior |
|---|---|
| Select user | Updates identity, role, access, login, and session details. |
| Activate / Deactivate | Opens a confirmation before changing account state. |
| Reset password | Requires a new password of at least 8 characters. |
| Reset 2FA | Opens a confirmation before removing the current second-factor setup. |
| Edit access | Opens role and feature controls for the selected user. |
| New User | Opens the user provisioning form. |

### Edit Access Dialog

**Edit access** can be opened from the row action button or from the selected-user panel. Role and feature switches update local dialog state first; **SAVE ACCESS** submits the selected roles and allowed features.

![System users edit access dialog](/images/vero-oms-ui/system-users-edit-access.png)

### Create User Dialog

**NEW USER** opens the create-user dialog. The form collects email, optional username, first name, last name, temporary password, role switches, and feature switches. The password visibility button only changes field display state. **CREATE USER** requires email plus a temporary password of at least 8 characters.

![System users create user dialog](/images/vero-oms-ui/system-users-create-user.png)

## Audit

Audit displays activity history, filter/search controls, refresh, and the selected record detail panel.

![System audit](/images/vero-oms-ui/system-audit.png)

| Audit control | Behavior |
|---|---|
| Search all columns | Filters the loaded audit entries by rendered step, request, actor, detail, and request id. |
| Actor filter | Filters activity by actor when refreshed or submitted with Enter. |
| Refresh | Reloads audit entries. |
| Row click | Opens the audit detail panel. |
| Related activity | Shows the timeline for entries that share the same request id. |

The detail panel displays the step, request title, actor, role, action, affected object, request id, changed values, and related activity when available.
