## Tabs and actions

<div className="guide-split guide-split--wide">
  <div className="guide-split-media">
    <img src="/images/vero-oms-ui/operation-tabs-actions.png" alt="Operation tabs and actions" />
  </div>
  <div className="guide-split-copy">

The action bar contains Accounts, Risk Rules, Approvals, search, New Group, Add Account, Reference Data, and Refresh.

| Area | Behavior |
|---|---|
| Accounts | Opens account mapping, group, access, and request workflows. |
| Risk Rules | Opens governed rule configuration and approval submission. |
| Approvals | Opens the maker-checker review queue. |
| Search | Filters account rows by account, name, email, phone, comp id, or group. |
| New Group | Creates a group directly when the user has permission. |
| Add Account | Opens a governed account-create request form. |
| Reference Data | Opens the account and counterparty reference values available to the user. |
| Refresh | Reloads the current operational data set. |

  </div>
</div>

## Account Table

<div className="guide-split guide-split--wide">
  <div className="guide-split-media">
    <img src="/images/vero-oms-ui/operation-account-table.png" alt="Operation account table" />
  </div>
  <div className="guide-split-copy">

The account table displays account id, name, type, group, margin, short sell, users, and rules.

Account rows can be sorted by account, name, type, group, margin, short-sell rule state, user count, and rule count. Row checkboxes enable bulk `ASSIGN USER` and `ADD TO GROUP`; each bulk action runs one governed operation per selected account and reports progress.

  </div>
</div>

## Account Detail

<div className="guide-split">
  <div className="guide-split-media">
    <img src="/images/vero-oms-ui/operation-account-detail.png" alt="Operation account detail" />
  </div>
  <div className="guide-split-copy">

The right-side account panel displays and manages the selected account.

| Detail action | Behavior |
|---|---|
| Edit Account | Opens an update-account request form prefilled from the selected account. |
| Deposit / Withdraw | Opens a request form for cash movement approval. |
| Add Rule | Opens a create-risk-rule request seeded to the selected account. |
| Move / Remove from group | Changes account group membership when allowed. |
| Grant / Revoke | Grants or revokes user access levels on the selected account. |
| Applied Rules | Displays active system/account rules affecting the selected account. |

Reference Data groups account and counterparty values by user-facing function. Users with manage permission can add values, confirm deletion, refresh the list, and close the dialog.

  </div>
</div>

## Risk Rules

<div className="guide-split guide-split--wide">
  <div className="guide-split-media">
    <img src="/images/vero-oms-ui/operation-risk-rules.png" alt="Operation risk rules" />
  </div>
  <div className="guide-split-copy">

Risk Rules contains the rule catalog, filters by engine/scope/kind, rules list, and rule detail panel.

| Risk rule action | Behavior |
|---|---|
| Search target rail | Filters rules by target or match value. |
| Category / scope / kind filters | Narrow the rule table to one rule family. |
| New Rule | Opens a maker-checker dialog for a new rule. |
| Select rule | Loads editable rule detail in the right panel. |
| Update Rule | Submits a change request; the rule does not apply until approved. |
| Delete Rule | Submits a delete request; removal does not apply until approved. |

  </div>
</div>

## Approvals

<div className="guide-split guide-split--wide">
  <div className="guide-split-media">
    <img src="/images/vero-oms-ui/operation-approvals.png" alt="Operation approvals" />
  </div>
  <div className="guide-split-copy">

Approvals contains review queues, request state, and the selected approval detail panel.

| Approval action | Behavior |
|---|---|
| All / Mine | Switches between all requests and requests submitted by the current user. |
| Status filter | Filters by pending, approved, applied, rejected, failed, or cancelled state. |
| Search | Searches id, request type, status, requester, or reviewer. |
| Select row | Opens request details, changed values, and review controls in the detail panel. |
| Approve / Reject | Available only to reviewers on other users' pending requests. |
| Cancel Request | Available to the requester while the request is pending. |
| Bulk approve / reject | Processes eligible pending requests and reports success/failure count. |

  </div>
</div>
